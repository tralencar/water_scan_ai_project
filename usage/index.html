    Project Usage

    This project includes automations via Makefile to simplify the execution of the main tasks. Below are the commands to run the classifier, check code quality, run tests, and update the documentation.

    üîπ Setting up the environment with Poetry

    Before running any command in this project, it is highly recommended to properly configure the Poetry virtual environment. This ensures that all dependencies are correctly installed and managed within the project.

    üîπ Create the virtual environment inside the project

    By default, Poetry creates the virtual environment outside the project directory. To keep the virtual environment within the repository, run:

    poetry config virtualenvs.in-project true

    üìå Notes:
    ‚û° This ensures that the virtual environment is created inside the .venv/ folder, making management easier.
    ‚û° Useful for maintaining isolated environments for each project.
    ‚û° Avoids conflicts between different dependency versions.

    üîπ Activate the virtual environment

    To activate the virtual environment and enter the Poetry shell, run:

    poetry shell

    üîπ Running the Water Scan AI Classifier

    To run the Water Scan AI classifier, execute:

    make run

    or directly via Poetry:

    poetry install

    poetry run python src/water_scan_main.py

    üìå Notes:
    ‚û° This command handles data loading, preprocessing, training, MLflow logging, and model registration.
    ‚û° It ensures that all dependencies are installed before execution.

    üîπ Checking Code Quality

    To run code quality checks using pre-commit, execute:

    make quality

    This command runs:

    • Black (automatic code formatting)
    • Isort (import sorting)
    • Flake8 (code linting)
    • Ruff (optimized linter)
    • Pre-commit hooks to prevent broken commits

    If you prefer to run it manually:

    poetry run pre-commit run --all-files

    üìå Notes:
    ‚û° Running make quality before committing helps prevent formatting issues and poor code quality.
    ‚û° Helps maintain a consistent code standard across the project.
    ‚û° Avoids CI/CD pipeline failures by catching errors early.

    üîπ Running MkDocs Documentation

    To preview the documentation locally:

    make doc

    or directly via Poetry:

    poetry run mkdocs serve

    Now, open http://127.0.0.1:8000/ in your browser.

    üìå Notes:
    ‚û° Useful for previewing the documentation before publishing.
    ‚û° Keeps the documentation always accessible to the team.
    ‚û° Prevents formatting issues and broken links before deployment.

    üîπ Publishing the Documentation Online (GitHub Pages)

    After making changes to the documentation, you can publish it online to GitHub Pages using the following command:

    poetry run mkdocs gh-deploy

    This will perform the following:
    1Ô∏è‚É£ Builds the documentation and converts the Markdown (.md) files to HTML.
    2Ô∏è‚É£ Creates a new commit on the gh-pages branch with the latest version of the documentation.
    3Ô∏è‚É£ Updates the public project page available on GitHub Pages.

    If the project is properly configured, the documentation will be available at:

    https://tralencar.github.io/water_scan_ai/

    üîπ Updating the Project Version (bump2version)

    If you want to bump the project version, use:

    poetry run bump2version patch   # For small changes
poetry run bump2version minor   # For minor updates
poetry run bump2version major   # For major updates

    This will automatically update the project version in pyproject.toml.

    üìå Notes:
    ‚û° Used to maintain version control as the project evolves.
    ‚û° Important for teams that follow semantic versioning (semver).
    ‚û° Ensures traceability of code changes.

    üîπ Running Tests (pytest)

    To run unit tests and check code coverage, execute:

    make test

    This command:

    • Installs development dependencies
    • Runs pytest with a minimum required coverage of 27%
    • Displays a coverage report

    üìå Notes:
    ‚û° Prevents broken code from reaching production.
    ‚û° Ensures correct functionality before adding new features.
    ‚û° The minimum coverage of 8% can be adjusted as more tests are added. This change must be made in the Makefile.

    üîπ Running Poetry Lock

    If you need to update the poetry.lock file, execute:

    make lock

    or directly via Poetry:

    poetry lock

    üìå Notes:
    ‚û° Useful when adding or updating packages in pyproject.toml.
    ‚û° Ensures that all dependencies are properly locked.
    ‚û° Prevents conflicts when working in teams.

    üîπ Managing MLflow with Docker Compose

    This project uses Docker Compose to run MLflow Tracking and MinIO services in an automated way.

    üîß Building the containers:

    make build

    üìå Note:
    ‚û° Builds and starts the services defined in mlflow-minio-setup/docker-compose.yml in the background.

    ‚õî Stopping the services:

    make end

    üìå Note:
    ‚û° Stops the running containers, such as MLflow and MinIO.

    üßπ Cleaning Up All Docker Containers:

    make clear_all

    üìå Notes:
    ‚û° Forcefully removes all stopped or running containers.
    ‚û° ‚ö† Use with caution, as it removes all local containers, including those from other projects.
